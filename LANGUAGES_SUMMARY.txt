8. REGEX (Regular Expressions)
   └─ Pattern-based vulnerability detection
   └─ Language: ECMAScript Regex
   └─ Purpose: Fast pattern matching in source code
   └─ Speed: Very Fast (< 1 second)
   └─ Example patterns:
      • SQL Injection: /\.query\s*\(\s*[`'"].*\$\{.*\}.*[`'"]\s*\)/g
      • XSS: /\.innerHTML\s*=\s*.*[\+$]/g
      • Command Injection: /(exec|execSync|spawn)\s*\([^)]*\$\{/g
      • Hardcoded Credentials: /(api[_-]?key)\s*[:=]\s*['"`][A-Za-z0-9]{20,}['"`]/gi

9. CODEQL (QL Language)
   └─ Semantic code analysis queries
   └─ Language: QL (CodeQL Query Language)
   └─ Purpose: Advanced vulnerability detection
   └─ Speed: Slower (10-60 seconds)
   └─ Supports: 10+ programming languages
   └─ Available Queries (10):
      • js/sql-injection (CWE-89)
      • js/xss (CWE-79)
      • js/command-line-injection (CWE-78)
      • js/code-injection (CWE-94)
      • js/hardcoded-credentials (CWE-798)
      • js/insecure-randomness (CWE-330)
      • js/path-injection (CWE-22)
      • js/xxe (CWE-611)
      • js/insecure-download (CWE-601)
      • js/prototype-pollution (CWE-1321)
